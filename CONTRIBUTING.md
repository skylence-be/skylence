# Contributing to Skylence

Thanks for your interest in Skylence.

## How to contribute

The source code for Skylence is private. Contributions happen through the issue tracker and discussions on this public repository.

### Reporting bugs

Use the [Bug Report](.github/ISSUE_TEMPLATE/bug.yml) template. Include your `sky --version` output, OS/platform, and reproduction steps. Error codes (`SKY-XX-NNN`) from logs are especially useful.

### Requesting features

Use the [Feature Request](.github/ISSUE_TEMPLATE/feature.yml) template. Describe the problem you're trying to solve, not just the solution you have in mind.

### Asking questions

Use [Discussions](https://github.com/skylence-be/skylence/discussions) for questions, ideas, and general feedback.

### Reporting security vulnerabilities

Do not open a public issue for security bugs. Report them privately via [GitHub Security Advisories](https://github.com/skylence-be/skylence/security/advisories/new). See [SECURITY.md](SECURITY.md) for details.

## Documentation

Documentation lives at [docs.skylence.be](https://docs.skylence.be). If you spot an error or gap, open an issue here and describe the problem.
